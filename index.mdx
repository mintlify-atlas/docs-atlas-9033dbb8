Typia is a TypeScript transformer library that generates runtime validation, serialization, and schema code directly from your TypeScript types at compile time. There are no decorators, no class inheritance, and no separate schema definitions — just pure TypeScript types.

## Why Typia?

Traditional validation libraries require you to maintain separate schema definitions alongside your TypeScript types. Typia eliminates that duplication — your types **are** the schema.

## Quick example

```typescript
import typia, { tags } from "typia";

interface IMember {
  id: string & tags.Format<"uuid">;
  email: string & tags.Format<"email">;
  age: number & tags.Minimum<18> & tags.Maximum<100>;
}

// One-line runtime validation — no extra schema needed
const member = typia.assert<IMember>(request.body);

// Type-safe JSON serialization
const json = typia.json.assertStringify<IMember>(member);
```

<Steps>
  <Step title="Install Typia">
    Add Typia to your project using your preferred package manager.
    ```bash
    npm install typia
    npx typia setup
    ```
  </Step>
  <Step title="Write TypeScript types">
    Define your data shapes as plain TypeScript interfaces — no decorators required.

  <Step title="Call Typia functions">

  </Step>
  <Step title="Run your build">
    Typia's transformer generates optimized runtime code at compile time — zero runtime overhead.
  </Step>
</Steps>

Typia's `misc` namespace provides three utility operations that go beyond basic validation: deep cloning objects as plain data, pruning superfluous properties to match a type exactly, and extracting all members of a union literal type into an array. Each operation has safe (no validation), assert, is, and validate variants so you can choose how strictly to handle type mismatches.

## `typia.misc.clone<T>()` — deep clone

`clone<T>` creates a deep copy of a value, stripping class methods and returning a plain `Resolved<T>` object. It does **not** validate the input — use the variants below for combined clone + validation.

```typescript
import typia from "typia";

interface IProfile {
  id: number;
  name: string;
  tags: string[];
  address: { city: string; zip: string };
}

const original: IProfile = {
  id: 1,
  name: "Alice",
  tags: ["admin"],
  address: { city: "Seoul", zip: "04524" },
};

const copy = typia.misc.clone<IProfile>(original);

// Mutations to copy do not affect original
copy.address.city = "Busan";
console.log(original.address.city); // "Seoul"
```

### The `Resolved<T>` return type

`clone` returns `Resolved<T>`, not `T`. `Resolved<T>` strips all methods from the type, converts class instances to plain objects, and unwraps boxed primitives (`Boolean` → `boolean`, etc.). Native types (`Date`, `Set`, `Map`) are preserved unchanged.

```typescript
class User {
  constructor(
    public id: number,
    public name: string,
  ) {}

  greet() {
    return `Hello, ${this.name}`;
  }
}

// Returns { id: number; name: string } — method is stripped
const cloned = typia.misc.clone<User>(new User(1, "Alice"));
// cloned.greet is undefined
```

### Clone variants

Choose the variant that matches your error handling strategy:

<Tabs>
  <Tab title="clone (no validation)">

```typescript
import typia from "typia";

// Deep clone without any type checking
const copy = typia.misc.clone<IProfile>(input);
```

Signature: `clone<T>(input: T): Resolved<T>`

  </Tab>
  <Tab title="assertClone">

```typescript
import typia from "typia";

// Throws TypeGuardError if input doesn't match T, then clones
const copy = typia.misc.assertClone<IProfile>(input);
```

Signature: `assertClone<T>(input: unknown, errorFactory?): Resolved<T>`

Throws `TypeGuardError` with `path`, `expected`, and `value` on first mismatch.

  </Tab>
  <Tab title="isClone">

```typescript
import typia from "typia";

// Returns null if input doesn't match T, otherwise returns clone
const copy = typia.misc.isClone<IProfile>(input);
if (copy === null) {
  console.error("Invalid input");
}
```

Signature: `isClone<T>(input: unknown): Resolved<T> | null`

  </Tab>
  <Tab title="validateClone">

```typescript
import typia from "typia";

// Returns IValidation with all errors, or cloned data on success
const result = typia.misc.validateClone<IProfile>(input);
if (result.success) {
  console.log(result.data); // Resolved<IProfile>
} else {
  console.error(result.errors); // IValidation.IError[]
}
```

Signature: `validateClone<T>(input: unknown): IValidation<Resolved<T>>`

  </Tab>
</Tabs>

### Factory functions

Use `createClone` and its variants when you need a reusable cloner for the same type:

```typescript
import typia from "typia";

const cloneProfile = typia.misc.createClone<IProfile>();
const assertCloneProfile = typia.misc.createAssertClone<IProfile>();
const isCloneProfile = typia.misc.createIsClone<IProfile>();
const validateCloneProfile = typia.misc.createValidateClone<IProfile>();

// Call repeatedly without re-generating the transform logic
const copies = profiles.map(cloneProfile);
```

## `typia.misc.prune<T>()` — remove extra properties

`prune<T>` **mutates** the input object in place, deleting any property not defined in type `T`, including in nested objects. This is useful when you receive data from an external source that may include undocumented fields.

<Warning>
`prune` mutates the input directly. Removed properties cannot be recovered. Clone the value first if you need the original.
</Warning>

```typescript
import typia from "typia";

interface IUser {
  id: number;
  name: string;
}

const raw = {
  id: 1,
  name: "Alice",
  password: "hunter2",     // extra — will be removed
  __debug: true,            // extra — will be removed
};

typia.misc.prune<IUser>(raw);

console.log(raw);
// { id: 1, name: "Alice" }
```

`prune` processes nested objects recursively:

```typescript
interface IAddress {
  city: string;
  country: string;
}

interface IOrder {
  id: string;
  address: IAddress;
}

const data = {
  id: "abc",
  address: {
    city: "Seoul",
    country: "KR",
    internalCode: "X92",   // removed from nested object
  },
  metadata: { trace: "xyz" }, // removed from root
};

typia.misc.prune<IOrder>(data);
// { id: "abc", address: { city: "Seoul", country: "KR" } }
```

### Prune variants

<Tabs>
  <Tab title="prune (no validation)">

```typescript
import typia from "typia";

// Mutates input: removes all properties not in T
typia.misc.prune<IUser>(input);
```

Signature: `prune<T extends object>(input: T): void`

  </Tab>
  <Tab title="assertPrune">

```typescript
import typia from "typia";

// Throws TypeGuardError if input type is wrong, then prunes
const result = typia.misc.assertPrune<IUser>(input);
```

Signature: `assertPrune<T>(input: unknown, errorFactory?): T`

Returns the pruned input typed as `T`. Throws `TypeGuardError` on mismatch.

  </Tab>
  <Tab title="isPrune">

```typescript
import typia from "typia";

// Returns false (no pruning) if type mismatch; true after prune
const ok = typia.misc.isPrune<IUser>(input);
if (!ok) {
  console.error("Input failed type check — not pruned");
}
```

Signature: `isPrune<T>(input: unknown): input is T`

  </Tab>
  <Tab title="validatePrune">

```typescript
import typia from "typia";

// Returns IValidation with all errors, or success after prune
const result = typia.misc.validatePrune<IUser>(input);
if (!result.success) {
  console.error(result.errors);
}
```

Signature: `validatePrune<T>(input: unknown): IValidation<T>`

  </Tab>
</Tabs>

## `typia.misc.literals<T>()` — extract union literal members

`literals<T>()` takes a union literal type and returns an array containing all its members. This is useful for populating dropdown options, generating validation allowlists, or iterating over a constrained set of string or numeric values.

```typescript
import typia from "typia";

type Status = "pending" | "active" | "suspended" | "deleted";

const statuses = typia.misc.literals<Status>();
// ["pending", "active", "suspended", "deleted"]
```

Works with numeric and mixed literal unions too:

```typescript
import typia from "typia";

type HttpCode = 200 | 201 | 400 | 401 | 403 | 404 | 500;
const codes = typia.misc.literals<HttpCode>();
// [200, 201, 400, 401, 403, 404, 500]

type Priority = "low" | "medium" | "high" | 1 | 2 | 3;
const priorities = typia.misc.literals<Priority>();
// ["low", "medium", "high", 1, 2, 3]
```

The return type is `T[]` where `T extends Atomic.Type | null` — meaning `boolean | number | string | bigint | null`.

<Note>
`literals<T>()` requires a type argument. It is a compile-time transform — the array is generated at build time, not by inspecting runtime values.
</Note>

### Real-world example: form select options

```typescript
import typia from "typia";

type Role = "admin" | "editor" | "viewer" | "guest";

// Get all valid options for a select element
const roleOptions = typia.misc.literals<Role>().map((value) => ({
  label: value.charAt(0).toUpperCase() + value.slice(1),
  value,
}));

// Validate that an incoming role string is a valid member
function isValidRole(input: unknown): input is Role {
  return (typia.misc.literals<Role>() as unknown[]).includes(input);
}
```

## Summary

| Function | Validates | Mutates | Returns |
|---|---|---|---|
| `clone<T>` | No | No | `Resolved<T>` (new object) |
| `assertClone<T>` | Yes (throws) | No | `Resolved<T>` |
| `isClone<T>` | Yes (null) | No | `Resolved<T> \| null` |
| `validateClone<T>` | Yes (errors) | No | `IValidation<Resolved<T>>` |
| `prune<T>` | No | Yes | `void` |
| `assertPrune<T>` | Yes (throws) | Yes | `T` |
| `isPrune<T>` | Yes (false) | Yes (if valid) | `boolean` |
| `validatePrune<T>` | Yes (errors) | Yes (if valid) | `IValidation<T>` |
| `literals<T>` | N/A | No | `T[]` |

<Warning>
All `typia.misc.*` functions are compile-time transforms. You must configure Typia's transformer in your build setup. See [Setup](/setup) for details.
</Warning>
